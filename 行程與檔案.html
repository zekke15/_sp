Unix 系統呼叫互動學習

fork  ·  execvp  ·  open  ·  read  ·  write  ·  close  ·  dup2  ·  stdin / stdout / stderr

核心概念

File Descriptor
fd（檔案描述符）
每個 process 都有一張「fd 表」，fd 是一個整數索引，指向系統中一個開啟的 I/O 端點（檔案、終端機、管線、socket…）。

你對 fd 做讀寫，核心就知道要去哪裡搬資料。

三個預設 fd（每個 process 啟動時就有）

數字名稱C 常數預設連接
0stdinSTDIN_FILENO終端機鍵盤（讀取）
1stdoutSTDOUT_FILENO終端機螢幕（寫入）
2stderrSTDERR_FILENO終端機螢幕（錯誤訊息）

fd 表示意圖

open() 回傳的 fd 從 3 開始往上遞增（0、1、2 已佔用）。

syscall
open()
int open(path, flags, mode)
開啟一個檔案，回傳 fd。
flags：O_RDONLY / O_WRONLY / O_RDWR / O_CREAT / O_TRUNC…
回傳值：成功 → 正整數 fd；失敗 → -1
syscall
close()
int close(int fd)
釋放 fd，讓核心知道你不再使用這個 I/O 端點。不 close 會導致 fd 洩漏。
回傳值：成功 → 0；失敗 → -1
syscall
read()
ssize_t read(fd, buf, count)
從 fd 讀最多 count bytes 到 buf。讀到 EOF 回傳 0，出錯回傳 -1。
fd 0 = 從鍵盤讀；也可以讀任何開啟的 fd。
syscall
write()
ssize_t write(fd, buf, count)
把 buf 的 count bytes 寫入 fd。
fd 1 = 寫到螢幕；fd 2 = 寫錯誤；任意 fd 均可。

範例：讀檔並印出

// 開檔 int fd = open("hello.txt", O_RDONLY); // fd 通常是 3（0、1、2 已佔） char buf[128]; // 從 fd 讀資料 ssize_t n = read(fd, buf, sizeof(buf)); // 寫到 stdout（fd 1） write(1, buf, n); // 釋放 fd close(fd);

生命週期

process
fork()
pid_t fork(void)
複製目前 process，建立一個幾乎一模一樣的子 process。兩個 process 從 fork() 之後的那行繼續跑。
回傳值：
· 父 process → 子的 pid（正整數）
· 子 process → 0
· 失敗 → -1
process
execvp()
int execvp(file, argv[])
用另一個程式取代目前 process 的記憶體映像，從頭執行新程式。fd 預設繼承！
成功：不回傳（舊程式已消失）
失敗 → -1

argv[0] 是程式名稱，最後一個元素必須是 NULL。

fork() 後的世界

典型程式碼模式

pid_t pid = fork(); if (pid == 0) { // 子 process char *args[] = {"ls", "-l", NULL}; execvp("ls", args); // 若 execvp 回來，代表失敗 perror("execvp"); exit(1); } else { // 父 process，等子結束 wait(NULL); }
關鍵細節：fork() 後，子 process 會繼承父 process 所有已開啟的 fd！這是 shell 管線和重導向的底層魔法。
syscall
dup2()
int dup2(int oldfd, int newfd)
把 oldfd 複製到 newfd。
如果 newfd 已開啟，會先 close 再覆蓋。
之後 oldfd 和 newfd 指向同一個開啟的檔案 / 管線。
最常用場景：把 fd 1（stdout）改為指向某個檔案，讓子 process 以為自己在寫螢幕，實際上寫進檔案。

stdout 重導向到檔案（ls -l > out.txt 的底層）

用 open("out.txt", O_WRONLY|O_CREAT|O_TRUNC, 0644) 取得 fd=3，指向檔案。
dup2(3, 1)：把 fd 3 複製到 fd 1。現在 fd 1 也指向 out.txt。
close(3)：不再需要多餘的 fd 3，避免洩漏。
execvp("ls", args)：ls 呼叫 write(1, ...)，資料流進 out.txt！

所有 syscall 速查表

syscall一句話說明
fork()建立子 process，子得 0，父得 child_pid
execvp()用新程式替換 process，fd 繼承
open(path, flags)開檔，回傳 fd
close(fd)關閉 fd，釋放資源
read(fd, buf, n)從 fd 讀最多 n bytes
write(fd, buf, n)寫 n bytes 到 fd
dup2(old, new)把 old fd 複製到 new fd（常用來重導向）
stdin = 0預設讀取端（鍵盤）
stdout = 1預設輸出端（螢幕）
stderr = 2預設錯誤輸出（螢幕）
